# 授業カレンダー連携の運用

More Better Gakujo の主導線は、アプリ内の「Googleスケジュール連携」から授業予定を追加・更新する方式です。ユーザーは `.ics` や購読URLを意識せず、連携方法、ターム判定、授業なし日の反映、追加先カレンダー名を同じ画面で確認して適用できます。

Android、iOS、macOSではOSカレンダーへ直接追加します。この経路では、公式授業暦のターム期間、授業なしの日、重複防止、一括削除を反映できます。予定のタイトル、場所、説明欄は、学務側のGoogle CalendarテンプレートURLに含まれる `text`、`location`、`details` だけを採用します。学務側のGoogle連携予定がページ上に見つからない場合は、More Better Gakujo側の推測で授業を生成せず、学務側の「Googleスケジュール連携」をフォールバックとして開きます。OSカレンダー直接追加が使えない環境では、最後の手段として `.ics` ファイルを書き出します。

ターム期間は、初期設定では新潟大学の公式授業暦PDFから現在日のタームを自動判定します。公式授業暦から取得できた授業なしの日は、設定で有効な場合に予定生成から除外します。必要に応じて、ページから読み取った期間または手入力の期間を使う設定にも変更できます。

## 重複を避ける仕組み

OSカレンダーへ追加する予定には、内部メモとして `MBG_UID:` から始まる識別子を付けます。再度追加・更新するときは、対象ターム期間内の More Better Gakujo 由来の予定を削除してから、最新の授業予定を入れ直します。

Android、iOS、macOSではOSカレンダーへ直接追加します。WindowsではOS共通の安全なローカルカレンダー書き込みAPIがないため、学務側のGoogle連携を優先し、それが見つからない場合は `.ics` を作成して既定のカレンダーアプリで開くフォールバックを使います。

## .ics 書き出し

`.ics` 書き出しは上級者向け、またはWindows向けのフォールバックです。ファイル名は `more-better-gakujo-classes.ics` に固定しています。`.ics` 内の各授業には `UID` を入れており、年度、ターム、授業名、曜日、時限、教室から作るため、同じタームの同じ授業を再書き出ししても同じイベントとして扱われやすくなります。

手動インポートを何度も行うと、カレンダーアプリによっては重複することがあります。重複を避けたい場合は、OSカレンダー直接追加を使うか、固定URLで購読する運用にしてください。

## GitHub Gist に置く

1. GitHub Gist で新しい Gist を作成します。
2. ファイル名を `more-better-gakujo-classes.ics` にして、アプリが書き出した内容を貼り付けます。
3. Gist の Raw URL をコピーします。
4. カレンダーアプリで「URLで追加」または「照会カレンダー」として追加します。
5. 次回以降は同じ Gist の同じファイルを上書きします。

Gist は secret にしても、URL を知っている人は内容を閲覧できます。授業名や教室が含まれるため、URL の共有範囲には注意してください。

## GitHub Pages に置く

1. GitHub Pages 用のリポジトリに `calendar/more-better-gakujo-classes.ics` を追加します。
2. GitHub Pages を有効化します。
3. 公開 URL、例 `https://ユーザー名.github.io/リポジトリ名/calendar/more-better-gakujo-classes.ics` をコピーします。
4. カレンダーアプリで URL 購読します。
5. 次回以降は同じパスのファイルを上書きします。

GitHub Pages は基本的に公開 URL になるため、授業予定を公開したくない場合は別の非公開ホスティング手段を使ってください。

## 主要カレンダーアプリでの追加先

- Google カレンダー: 設定の「カレンダーを追加」から「URLで追加」
- Apple カレンダー: macOS は「ファイル」から「新規照会カレンダー」、iOS/iPadOS は設定アプリのカレンダーアカウント追加
- Outlook: 「カレンダーを追加」から「Webから追加」

反映間隔はカレンダーアプリ側の仕様に依存します。ファイルを更新してもすぐ反映されない場合があります。
